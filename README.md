# FreightCast

FreightCast 是一個前端靜態專案，用來根據真實環境資料與規則模型，估算國際貨運的可能延遲天數。

目前支援：
- 空運與海運兩種模式
- 指定出發地與目的地城市
- 即時天氣與未來短期預報
- 國定假日與週末影響
- 季節性風險
- 航線咽喉點風險
- 分析摘要複製

## 專案特色

- 不需要建置工具，直接用瀏覽器開啟即可執行
- 使用免費公開 API 取得天氣與節假日資料
- 以基線運輸時間加上多模組風險分數，產出最終估算天數
- 針對跨時區假日判斷做了地區化修正
- 已避免長程路線直接使用目的地未來 3 天天氣而造成高估

## 使用方式

1. 開啟 `index.html`
2. 選擇出發地與目的地
3. 選擇空運或海運
4. 點擊「啟動環境數據分析」
5. 查看基線時間、風險因素、綜合延遲與最終預估總耗時

如果要在本機用較穩定的方式開啟，也可以使用任何靜態伺服器，例如 VS Code Live Server。

## 判斷邏輯

核心邏輯位於 [script.js](/abs/path/c:/Users/btohr/Desktop/AI_text/script.js)。

### 1. 基線運輸時間

系統先依據：
- 運輸方式：`air` / `sea`
- 區域組合：例如 `Asia_Europe`、`NA_NA`

取得一組基線時間區間，例如：
- 亞洲到歐洲空運：3-5 天
- 亞洲到歐洲海運：25-35 天

### 2. 天氣延遲

天氣模組會分析：
- 即時降雨量
- 即時風速
- WMO weather code
- 未來 3 天預報

規則特性：
- 空運對雷暴、冰雹、濃霧、強風更敏感
- 海運對高風速、暴風雨、港口作業受阻更敏感
- 長程路線的目的地，不再直接把未來 3 天預報加進分數，避免結果失真

### 3. 假日與週末延遲

假日模組會分析：
- 出發地當地日期是否為週末
- 出發地是否為週五出貨
- 出發地與目的地是否為國定假日
- 未來 3 天是否有連假

這部分目前已改成依城市自己的 `timezone` 判斷，而不是直接使用使用者瀏覽器所在地時間。

### 4. 季節性風險

依月份與區域給予額外風險，例如：
- 亞洲颱風季
- 北美冬季暴風雪
- 歐洲冬季風暴
- 全球物流旺季
- 亞洲農曆新年期間

### 5. 航線咽喉風險

海運模式下，會對特定區域組合加入固定風險，例如：
- 蘇伊士運河
- 巴拿馬運河
- 荷莫茲海峽

## API 金鑰欄位說明

畫面中有一個「專業物流 API 金鑰」欄位，這個欄位目前是為未來擴充預留。

目前行為：
- 若未輸入金鑰，系統只使用免費公開資料與內建規則
- 若輸入金鑰，系統會額外套用示意性加權
- 這段加權目前不是實際串接商業物流 API 的結果，而是保留未來整合位置

未來若要正式串接，可參考：
- AviationStack 註冊頁面
  - https://aviationstack.com/signup/free
- Terminal49 API Quickstart
  - https://terminal49.com/docs/api-docs/in-depth-guides/quickstart/

建議之後若要正式接入：
- 將 API 回傳的壅塞、航班、港口或追蹤資料轉成可解釋的加權來源
- 不要再使用隨機值作為加權依據

## 已知限制

- 目前仍是規則式估算器，不是真正的物流追蹤系統
- 季節性與航線風險仍屬固定加權，未直接連動即時全球事件
- API 金鑰欄位尚未接入真實商業服務
- 未使用後端，因此所有邏輯都在前端執行
- 若公開 API 暫時無回應，部分結果會退回較保守或較簡化的顯示

## 後續可強化方向

- 將 API 金鑰欄位真的接上航班、港口或貨櫃追蹤服務
- 為假日模組設定加權上限，避免連續條件過度疊加
- 依基線時間推估 ETA，使用更貼近到港日的目的地天氣
- 加入更多城市、港口與機場
- 為規則計算加入測試案例
